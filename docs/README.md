## Documentation Standards

This documentation follows PKF standards:

- **Navigation hubs** - Every directory has a README.md
- **Consistent naming** - SCREAMING-KEBAB-CASE for specs, kebab-case for guides
- **Register format** - YAML frontmatter in register entries
- **Cross-references** - Relative paths for internal links
